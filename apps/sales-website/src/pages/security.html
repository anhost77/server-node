Security Architecture

Ed25519 Cryptographic Signatures

Every command sent through ServerFlow is cryptographically signed using Ed25519, the same elliptic curve cryptography used by Signal, OpenSSH, and TLS 1.3. This mathematically guarantees that commands cannot be forged or tampered with.

Outbound-Only Connections

The ServerFlow agent on your server initiates all connections to our orchestrator. We never connect to your server. This means:

  • No SSH keys to manage or leak
  • No open ports required on your firewall
  • No way for us to access your server directly

Zero Credential Storage

We don't store your server credentials. Period. If ServerFlow's infrastructure is compromised, attackers find nothing they can use to access your servers. Your keys live only on your servers.

Open Source Agent

Our agent is 100% open source. You can audit every line of code that runs on your server. We encourage security researchers to review our code and report any vulnerabilities through our responsible disclosure program.

Data Privacy

ServerFlow processes minimal data: deployment commands and basic telemetry. We never access your application code, database contents, or user data. All data transmission is encrypted using TLS 1.3.
